# Zhipu Image Generation

Use the Zhipu GLM-Image model to generate AI illustrations and embed them into the HyperFrames HTML composition.

**Scope of responsibility:** Handles image concerns only — prompt quality, API calls, image density, and size adaptation. It does not cover HTML layout, animation, or template design (those are the responsibility of `/hyperframes`).

## Prerequisites

- The project root `.env` file contains `ZHIPU_API_KEY` (obtain at: https://open.bigmodel.cn)
- Node.js 18+ (for the helper scripts)

## Workflow

### Scene-driven mode (when material-requests.json exists)

When `material-requests.json` (generated by hyperframes) exists in a project, image generation is driven by the `imageSlots` array, skipping the manual flow.

**The user only needs to call `/image-gen`, with no extra arguments.**

#### Auto-read data

1. Read `material-requests.json` → get the `imageSlots` array
2. Read `brief.json` → get `visualStyle` (used to unify the visual style)
3. Read the HTML composition (if present) → confirm image container sizes

**Format of imageSlots in material-requests.json:**
```json
{
  "imageSlots": [
    {
      "slotId": "img-scene-003",
      "sceneId": "scene-003",
      "prompt": "7-element image prompt (Chinese, 2-3 complete sentences)",
      "containerSize": { "width": 800, "height": 600 },
      "style": "Shadow Cut"
    }
  ]
}
```

**Note:** The prompt in imageSlots is already a complete prompt processed through the 7-element method and does not need further expansion.

#### Prompt quality check

Before calling the API, check that each imageSlot's prompt meets the quality bar:

- At least 2-3 complete sentences
- Contains specific objects, materials, environment, lighting, colors, mood, composition
- Prefer Chinese for the actual GLM-Image call (the model yields higher quality for Chinese prompts)
- Consistent with the visualStyle from brief.json

If a prompt is too short or abstract (fewer than 30 characters), expand it with an LLM (see the "LLM expansion for abstract text" section below).

#### Image size handling

The `containerSize` in imageSlots may not be a standard size. Convert it to an API-supported size:

```javascript
// containerSize → API size conversion rules
// 1. Find the closest standard size matching the containerSize aspect ratio
// 2. If no standard size matches, use a custom size: align to a multiple of 32, no larger than 2048px
// 3. The generated image only needs to be slightly larger than the container (object-fit: cover will crop)
```

**Standard size table:**

| Ratio | Standard sizes |
|------|---------|
| 1:1 | 1280x1280, 1536x1536 |
| 16:9 | 1728x960, 1472x832 |
| 9:16 | 960x1728, 832x1472 |
| 4:3 | 1568x1056, 1472x1104 |
| 3:4 | 1056x1568, 1104x1472 |

#### Generate images

Convert imageSlots to a batch JSON and call generate-image.mjs:

```bash
# Build batch prompts from material-requests.json
node skills/image-gen/scripts/generate-image.mjs \
  --batch image-batch.json \
  --output-dir <composition-dir>/assets/ \
  --no-watermark
```

`image-batch.json` format (auto-converted from material-requests.json):
```json
[
  { "prompt": "from imageSlots[0].prompt", "filename": "img-scene-003.png", "size": "1280x960" },
  { "prompt": "from imageSlots[1].prompt", "filename": "img-scene-005.png", "size": "1280x1280" }
]
```

#### Update HTML

After image generation completes, confirm the corresponding `<img>` `src` in the HTML composition points to the correct path:
- `src="assets/<slotId>.png"`
- The filename matches the slotId in material-requests.json imageSlots

#### If imageSlots is empty

material-requests.json may have no imageSlots (all scenes are CSS-drawn or pure text). In that case, inform the user:

```
material-requests.json contains no scenes that require AI generation (all visuals are drawn with CSS/SVG or presented as pure text).
No need to call image generation.
```

### Standalone mode (when no material-requests.json exists)

When `material-requests.json` does not exist in the project, use the full manual flow.

#### Step 1: Determine image density

**One image per knowledge point.** Video is a visual medium — if a concept is important enough to mention, it is important enough to illustrate.

**Image density rule: one image per knowledge point.**

| Video duration | Typical scene count | Recommended image count |
|---------|-----------|-----------|
| 15-30s | 4-6 | 3-5 |
| 30-60s | 7-10 | 6-9 |
| 60-120s | 12-18 | 10-16 |

**Rules:**
- Every knowledge point must have an image. If the script covers 5 concepts, generate 5 images.
- Title scenes — no image. Ending scenes — no image.
- Each sub-point gets its own image (e.g. one each for "drug discovery" and "cryptography")
- Scenes shorter than 2 seconds do not get images
- Content-dense technical content should be split into more scenes

#### Step 2: LLM expansion for abstract text

Knowledge points in a script are usually abstract — "quantum superposition" or "drug discovery" are concepts, not visual scenes. **Feeding abstract text directly to image generation produces empty, generic images.** First use an LLM to expand them into concrete, vivid visual language.

**When to expand:**
- The source text is a concept, theory, process, or domain term (not a visual description)
- After reading the text you cannot immediately picture a concrete scene
- The text is shorter than one complete visual description sentence

**When not to expand:**
- The text already describes a concrete visual scene (with specific objects, environment, details)
- The text comes from a detailed script that has already been visualized

**Expansion method — LLM prompt:**
```
You are a visual description expert. Expand the following abstract concept into a concrete visual description usable for AI image generation.
Requirement: describe specific objects, materials, environment, lighting, colors, and composition, not abstract concepts.
Do not explain the concept itself; instead, depict a picture that can be "seen".

Concept: [source text]
```

**Expansion quality check — the output should:**
- Name **specific objects** (not "technology" → "gold flip-chip on a copper pillar")
- Specify the **environment** (not "laboratory" → "deep-blue industrial cryogenic lab, dense blue coaxial cables")
- Describe the **lighting** (not "bright" → "soft volumetric light reflecting off the chip surface")
- Include **color and mood** (not "tech feel" → "cool cyan tones, precise and calm atmosphere")
- Be at least 2-3 sentences

**Before-vs-after LLM expansion:**

| Source text | Direct to API (poor) | After LLM expansion (good) |
|--------|---------------|-----------------|
| Quantum entanglement | "Quantum entanglement phenomenon" | "two glowing quantum particles linked by a golden thread in deep space, surrounded by purple and blue nebula gas, with energy ripples and micro-ring halos flowing across their surfaces" |
| Drug discovery | "Quantum drug discovery" | "a 3D pharmaceutical molecule structure slowly rotating on a dark holographic screen, atomic nodes connected by colored light-bond sticks, surrounded by translucent protein-folding outlines" |
| Quantum computing applications | "Applications of quantum computing" | "a collage of four futuristic-city scenes — a giant digital lock being cracked, optimized logistics routes, a multi-layer neural network of glowing connected nodes, and a waterfall of financial data" |

The expanded text proceeds to Step 3 (web search to supplement concrete details) and Step 4 (7-element prompt construction).

#### Step 3: Web search for visual details

An overly abstract prompt produces generic footage. **All professional, technical, or specialized topics must be web-searched.**

**Cases that must be searched:**
- The topic contains **any** technical, scientific, medical, financial, or domain-specific term
- The topic is a concept, theory, algorithm, or process (not a concrete object like "apple" or "office")
- You cannot immediately picture concrete visual details in your mind

**Search strategy:**
1. Search `[topic name] visualization OR illustration OR concept diagram`
2. Search `[topic name] real-world application OR case study`
3. If results are vague, search `[topic name] practice` or `[topic name] diagram`
4. Extract **concrete visual elements**: colors, objects, environment, textures, composition

**Extract the 7 elements for each image** (based on the official Zhipu guide):

| Element | Question | "Quantum superposition" example |
|------|------|---------------|
| **Subject** | Central visual object? | Translucent Bloch sphere, superposition projection on its surface |
| **Medium** | Photography, painting, illustration, 3D? | 3D concept art / high-definition photography |
| **Environment** | Where? | Deep-blue quantum lab, lab bench |
| **Lighting** | What light? | Volumetric light, cool-toned glow, precise tech feel |
| **Color** | Color palette? | Deep blue + cyan glow + purple shadows |
| **Mood** | What atmosphere? | Mysterious, precise, cutting-edge tech |
| **Composition/angle** | Camera angle? | Top-down 45°, sphere centered |

**Association technique — when information is insufficient:**

Link the abstract topic to a concrete metaphor or analogy:

| Abstract topic | Association | Concretized visual |
|---------|------|-----------|
| Decoherence | Shattered glass sphere in static noise | A crystalline qubit glass sphere with cracks forming on its surface, red warning light, noise ripple impact |
| Quantum advantage | Absolute lead in a race | Quantum computer as a light-speed starship, classical computer as a propeller plane, speed comparison shot |
| Shor's algorithm | Brute-forcing a combination lock | On a giant wall of digital passwords, key fragments fly out of a quantum circuit and assemble |

#### Step 4: Build the prompt

Based on the 7 elements from Step 3, build a **structured, detailed** prompt for GLM-Image.

**Prompt quality rules (official Zhipu guide):**
- Use **precise, specific visual descriptions**, not abstract concepts
- Include **subject + medium + environment + lighting + color + mood + composition**
- Prefer Chinese for the actual GLM-Image call — the model yields higher quality for Chinese prompts (the examples in this doc are written in English only for consistency)
- Be rich and descriptive — avoid one-line generic prompts

**Prompt formula:**
```
[concrete subject + material details], [medium/art style], [concrete environment + props], [lighting description], [colors/tones], [mood/atmosphere], [composition/angle]
```

**Good prompt example:**
```
a translucent Bloch sphere hovering over the lab bench of a deep-blue quantum laboratory,
3D concept-art style, surrounded by glowing qubit lines and precision optical instruments,
volumetric light shining from inside the sphere, deep blue and cyan as the dominant tones
with purple shadows, a mysterious and precise high-tech mood, 45° top-down composition.
```

**Poor prompt (too abstract):**
```
quantum superposition
```

**If the prompt is still thin**, return to Step 3 to search for more concrete visual references.

#### Step 5: Call the API to generate images

Use the helper script to call the Zhipu API. The script reads `ZHIPU_API_KEY` from the project root `.env`.

**Single image:**
```bash
node skills/image-gen/scripts/generate-image.mjs \
  --prompt "your prompt here" \
  --output compositions/assets/generated-NAME.png \
  --size "1728x960" \
  --no-watermark
```

**Batch mode** — generate multiple images in parallel, with support for a **custom size per image**:
```bash
node skills/image-gen/scripts/generate-image.mjs \
  --batch prompts.json \
  --output-dir compositions/assets/ \
  --no-watermark
```

`prompts.json` format:
```json
[
  { "prompt": "prompt 1", "filename": "scene-hero.png", "size": "1728x960" },
  { "prompt": "prompt 2", "filename": "scene-data.png", "size": "1280x1280" },
  { "prompt": "prompt 3", "filename": "scene-detail.png" }
]
```

The `size` field on each entry is optional; when omitted, the `--size` default (`1280x1280`) is used.

**Available sizes:** `1280x1280` (default), `1568x1056`, `1056x1568`, `1472x1088`, `1088x1472`, `1728x960`, `960x1728`
Custom: 1024-2048px per side, total pixels no more than 2^22, a multiple of 32.

**Notes:**
- The API is asynchronous — after submitting a task, poll until it completes (about 20 seconds for HD quality)
- Use batch mode to generate multiple images in parallel when needed
- Images are saved locally; the API's temporary URL expires after 30 days
- When HD quality is not needed, set `--quality standard` to speed it up (about 5 seconds)

#### Step 6: Handoff with Hyperframes

This skill alternates with the hyperframes skill in the flow:

```
image-gen  Step 1-4  →  outputs the prompt list
        ↓
hyperframes        designs the HTML layout  →  determines the aspect ratio of each image container
        ↓
image-gen  Step 5      →  generates images at the given ratios
        ↓
hyperframes        assembles the final HTML + animation + render
```

**image-gen → hyperframes (after Step 4):**
- The prompt list (one per scene)
- The content type of each scene (concept explanation, data statistics, process steps, list, abstract concept, etc.)
- Which scenes have images and which do not (title/ending)

**hyperframes → image-gen (before Step 5):**
- The aspect ratio of each image container
- The mapping between image filenames and scenes

**image-gen → hyperframes (after Step 5):**
- File paths of the generated images
- The actual generated size of each image

This skill does not participate in HTML layout design — that is entirely the responsibility of hyperframes.

## Image technical rules

- **The image aspect ratio must match the container** — generate at the ratio the layout requires
- All images use `object-fit: cover` and declare `aspect-ratio` — do not use `height: auto`
- Add `crossorigin="anonymous"` to all `<img>` tags
- Add `class="clip"` to images that use `data-start`/`data-duration`
- All images within the same composition should share a consistent visual style
- Use `box-shadow` to give images a "grounded" feel on dark backgrounds

## Common issues

| Issue | Solution |
|------|---------|
| `ZHIPU_API_KEY` not found | Add `ZHIPU_API_KEY=your_key` to the project root `.env` |
| Task stuck in PROCESSING | Wait a bit longer — HD quality takes about 20 seconds. Use `--quality standard` to speed it up |
| Poor image quality | Improve the prompt — add more specific style/descriptive keywords |
| Repetitive scene footage | Layout and HTML structure are hyperframes' concern — make sure different content types were communicated |
| Images feel static | Ambient motion on images is hyperframes' concern — communicate this need to it |
| Watermark still present | On the Zhipu platform, go to User Center → Security Management → Watermark Removal and sign the liability waiver |
